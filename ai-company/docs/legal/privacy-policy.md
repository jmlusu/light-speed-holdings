## 1. Introduction

This Privacy Policy describes how Light Speed Holdings ("Company," "we," "us," or "our") collects, uses, stores, and protects information when you ("User," "you," or "your") use the AI Company Builder software and related services (collectively, the "Service"). We are committed to protecting your privacy and being transparent about our data practices.

This Privacy Policy applies to all users of the Service, including individuals and organizations using the AI Company Builder CLI, dashboard, and related tools.

## 2. Information We Collect

### 2.1 Agent Configuration Data

When you use the Service, you create agent definitions that describe your AI agent hierarchy:

- **Agent identifiers**: IDs, names, titles, department assignments
- **Agent roles**: Responsibilities, reporting relationships, seniority levels
- **Tool permissions**: Which tools each agent can access (read, write, execute, etc.)
- **Agent instructions**: System prompts, behavioral guidelines, mission statements

**How we use it**: This data is stored locally in your `company-registry.yaml` file and is processed only on your system. We do not have access to your agent configurations unless you explicitly share them with us.

### 2.2 Task Data

The Service processes tasks through the MessageBus system:

- **Task instructions**: Text prompts and instructions sent to AI agents
- **Task results**: Outputs generated by AI agents in response to tasks
- **Task metadata**: Status, priority, timestamps, sender/receiver IDs
- **Task history**: Record of completed and pending tasks

**How we use it**: Task data is stored locally in `.opencode/inbox.json` and processed on your system. Task data may be sent to third-party LLM providers (see Section 2.4) for processing.

### 2.3 LLM Usage Logs

The CostTracker component records detailed logs of all LLM API calls:

- **Timestamp**: Date and time of each LLM call
- **Model identifier**: Which LLM model was used (e.g., `gpt-4o`, `claude-3-5-sonnet`)
- **Provider identifier**: Which LLM provider processed the call (e.g., `openai`, `anthropic`)
- **Agent name**: Which agent made the call
- **Task ID**: Which task the call was associated with
- **Token counts**: Number of prompt and completion tokens
- **Cost**: Calculated cost in USD based on published pricing
- **Iteration number**: Which iteration of the agentic loop produced the call

**How we use it**: Usage logs are stored locally in `results/cost_log.jsonl`. They are used for budget enforcement, cost reporting, and optimization. We do not have access to your usage logs unless you explicitly share them.

### 2.4 Data Sent to LLM Providers

When you configure the Service to use cloud-based LLM providers (OpenAI, Anthropic, DeepSeek), the following data is transmitted to those providers:

- **System prompts**: Agent instructions and context
- **User prompts**: Task instructions and conversation history
- **Tool outputs**: Results of tool executions (file contents, command outputs)

**Important**: This data is sent directly from your system to the LLM provider. Light Speed Holdings does not intercept, store, or have access to this data in transit.

**Local processing option**: If you use Ollama for local model inference, no data is transmitted to external servers. All processing occurs on your local machine.

### 2.5 Dashboard Data

The CEO Dashboard displays operational metrics:

- **KPI metrics**: Aggregated data about task completion, costs, and agent performance
- **System health**: Status of system components (MessageBus, CostTracker, etc.)
- **Agent status**: Which agents are active and processing tasks

**How we use it**: Dashboard data is derived from your local data (task history, usage logs) and is displayed only on your local system. The dashboard does not transmit data to external servers.

### 2.6 Contact Information

If you contact us for support or provide feedback:

- **Email address**: If you email us directly
- **Name**: If you provide it in correspondence
- **Organization**: If you identify your employer or organization
- **Feedback content**: Any information you choose to share

**How we use it**: Contact information is used solely to respond to your inquiry and is not shared with third parties.

## 6. Data Security

### 6.1 Technical Measures

We implement the following security measures to protect data processed by the Service:

| Measure | Implementation | Scope |
|---------|---------------|-------|
| Path sandboxing | `ToolRunner._safe_path()` restricts file access to project root | All file operations |
| HITL gates | Dangerous tools (write, execute, code_interpreter) require human approval | Tool execution |
| Budget enforcement | `CostTracker.check_budget()` prevents runaway LLM spend | LLM API calls |
| API key protection | Keys stored in environment variables, never in source code | Authentication |
| Provider fallback | Automatic failover on provider errors | LLM availability |

### 6.2 LLM Provider Security

When data is transmitted to cloud-based LLM providers:

- Data is encrypted in transit (TLS 1.2+)
- Each provider implements their own security measures
- API authentication is required for all requests
- Rate limiting and access controls are enforced

### 6.3 Local Processing Security

When using Ollama for local inference:

- All data remains on your local machine
- No network transmission occurs
- You control the security of your local environment
- You are responsible for securing your local installation

### 6.4 Security Incident Response

In the event of a security incident affecting your data:

1. We will notify affected users within 72 hours
2. We will investigate the incident and take corrective action
3. We will cooperate with relevant authorities as required by law
4. We will document the incident and implement preventive measures

## 13. Data Processing Summary

For quick reference, here is a summary of how data flows through the Service:

```
User Input
    |
    v
Agent Configuration (company-registry.yaml)
    |
    v
Task Creation (MessageBus: .opencode/inbox.json)
    |
    v
Agent Loop Execution (AgentLoop)
    |
    +---> LLM Provider (OpenAI/Anthropic/DeepSeek) [external transmission]
    |         |
    |         v
    |     LLM Response
    |
    +---> Tool Execution (ToolRunner) [local only]
    |         |
    |         v
    |     Tool Results
    |
    +---> Cost Tracking (CostTracker: results/cost_log.jsonl) [local only]
    |
    v
Task Completion
    |
    v
Dashboard Display (FastAPI: local server) [local only]
```

**External data transmission**: Only occurs when using cloud-based LLM providers. All other data processing occurs locally on your system.
